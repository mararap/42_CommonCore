This tester allows you to test:

	a. Invalid input handling
	- checks if the program handles invalid input correctly.

	b. when philosophers should die
	- checks if the program stops when a philosopher dies in the expected time.

	c. when no philosophers should die
	- checks if the program runs for a certain time without any philosopher dying.
	 with and without number of meals limitation.

	d. check for data races and deadlocks
	- using helgrind, drd, and sanitizer.

	e. check timestamps
	- checks if the program runs for a certain time without mixing timestamps

	f. The Chaos Feast
	- runs all tests in sequence.


Choose test to run:	
  [1] check invalid input (auto)	
  [2] die tests - auto (can take a while)	
  [3] no-die limited meals test (auto)	
  [4] no-die tests (can take a while)	
  [5] no-die tests (auto)	
  [6] check data races && deadlocks (needs docker) 	
  [7] check timestamps - The Chefs Special 	
  [8] The Chaos Feast (auto)	
  [ESC] exit tester
